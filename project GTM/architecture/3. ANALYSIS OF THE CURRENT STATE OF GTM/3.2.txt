────────────────────────────────────────────────────────────────────────────────────
3.2.3. TECHNICAL CONSTRAINTS AND BOTTLENECKS
────────────────────────────────────────────────────────────────────────────────────

1. **Lack of real‑time synchronization** – data from Snowflake to Salesforce
   is updated with a delay of up to several hours (batch mode).
2. **Manual export/import** – intent signals from 6sense are not automatically
   integrated into CRM; Sales Strategists spend time processing Excel.
3. **No event‑driven architecture** – changes in CRM do not trigger automatic
   actions in other systems (e.g., when an Opportunity is created, a sequence
   in SalesLoft is not launched).
4. **API limits** – Salesforce API limits may be exceeded during mass updates,
   requiring batch processing.
5. **Fragmented data sources for CS** – Gainsight is not yet integrated with
   the product and support in real time.

────────────────────────────────────────────────────────────────────────────────────
3.2.4. DEVELOPMENT POTENTIAL
────────────────────────────────────────────────────────────────────────────────────

- Salesforce Platform Events can be used for event‑driven integration.
- Hightouch supports real‑time synchronization via webhooks.
- 6sense has a REST API that can be used for automatic export of intent data.
- Gainsight allows configuration of playbooks and automated tasks.

────────────────────────────────────────────────────────────────────────────────────
**PROJECT CONCLUSIONS**
- The existing stack allows building an event‑driven architecture without replacing
  tools, but requires configuring API integrations and implementing an orchestration
  platform (AWS EventBridge, agent holding).
- Key bottlenecks: lack of automatic intent data synchronization, manual data
  exchange between Snowflake and Salesforce, and unautomated playbooks.
